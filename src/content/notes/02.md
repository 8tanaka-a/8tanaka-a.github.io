バックエンドの技術選定で、NestJS と Drizzle ORM の組み合わせに落ち着きました。
アプリの要件を構造で支えられるかどうか、を軸に考えました。

NestJS は、DI のリクエストスコープでコンテキストを透過的に扱えるのが気に入っています。
Guard と Repository の基底クラスで分離を強制できるので、
実装漏れを構造側で受け止めてくれる感覚があります。

Drizzle は、SQL に近い記述で発行クエリが読みやすい。
RLS の SET LOCAL 運用やデバッグもしやすそうで、自分の用途には合っていると感じています。

TypeScript で FE/BE を統一できる恩恵もあって、構造の強さで選んだ形です。
