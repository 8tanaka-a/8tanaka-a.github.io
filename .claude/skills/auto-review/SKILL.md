# Auto Review

ステージ済みの変更差分に対してレビューを行い、問題があれば修正→再レビューを繰り返す。
最大3ラウンド。対応すべき問題がなくなった時点で終了する。

## レビュー手順

各ラウンドで以下を実行する:

1. `git diff --cached` でステージ済みの差分を取得
2. `pnpm lint` と `pnpm format:check` を実行
3. 差分を以下の観点でレビューする

## レビュー観点

- **アクセシビリティ**: セマンティックHTML、aria属性、キーボード操作、alt属性、コントラスト比 WCAG AA
- **パフォーマンス**: 不要なレンダリング、未最適化な画像/フォント、バンドルサイズ
- **セキュリティ**: XSS、インジェクション、機密情報のハードコード
- **コード品質**: 型安全性、未使用コード、エラーハンドリング
- **プロジェクト規約**: CLAUDE.md の方針に準拠しているか

## 分類

各指摘を以下のいずれかに分類する:

- **CRITICAL**: バグ、セキュリティ脆弱性、アクセシビリティ違反など、必ず修正すべき問題
- **WARNING**: パフォーマンス懸念、ベストプラクティス違反など、対応すべき問題
- **INFO**: 改善提案、スタイルの好みなど。修正は不要

## ラウンドの流れ

1. レビューを実施し、指摘事項を一覧で報告する
2. CRITICAL または WARNING がある場合、それらを修正する
3. 修正後、次のラウンドで再レビューする
4. CRITICAL と WARNING がすべて解消されたら終了する
5. 最大3ラウンドまで。3ラウンドで解消しきれない場合は残課題を報告して終了する

## 出力フォーマット

各ラウンドの報告は以下の形式で出力する:

```
## Review Round N/3

### CRITICAL
- [ファイル:行] 内容

### WARNING
- [ファイル:行] 内容

### INFO
- [ファイル:行] 内容

### Result: PASS / FIXING
```
