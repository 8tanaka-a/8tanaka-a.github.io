# Note 作成

ユーザーと対話しながら Notes の記事を作成し、`src/content/notes/` に Markdown ファイルとして保存する。

## 流れ

1. **テーマの確認**: ユーザーに何について書きたいか聞く（引数があればそれをテーマにする）
2. **種別の確認**: 記録系（#）か調べた系（?）かを確認する
3. **文章の壁打ち**: テーマをもとに文章のたたき台を提案する。ユーザーのフィードバックを受けて修正を繰り返す
4. **タイトルの決定**: 本文が固まったらタイトルを提案し、確定する
5. **ファイル生成**: タイトル・本文・種別が揃ったら Markdown ファイルを作成する

## 文章のスタイル

- **本文は300〜350文字程度**（4〜5段落）
- one code のトーンに合わせる: 飾らない、素直、短文ベース
- 段落は短く。1段落2〜3文程度
- 技術的な正確さは保ちつつ、個人の感想・気づきを交える
- 「〜だと思います」「〜な気がします」のような柔らかい語尾を適度に使う
- 既存の About ページや Works の文体を参考にする

## ファイル名規則

- `{連番2桁}.md` — 例: `01.md`, `02.md`
- `src/content/notes/` 内の既存ファイルの最大番号 + 1

## テンプレート

```markdown
---
title: { タイトル }
date: { 今日の日付 YYYY-MM-DD }
dotPattern: { hash or question }
---

{本文}
```

## 注意事項

- 文章が固まるまでファイルは作成しない。ユーザーが「これでOK」と言ったら生成する
- `date` は作成日を自動で入れる
- `dotPattern` は `hash`（記録系 #）または `question`（調べた系 ?）
- ファイル作成後、パスと `/notes/{id}` の URL を表示する
